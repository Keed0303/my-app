# Semantic Commit Guidelines

This document outlines the standard for commit messages used by the team to ensure clarity and consistency.

## Commit Types

- **feat**: A new feature or addition  
  Example: `feat: add user login functionality`

- **fix**: A bug fix  
  Example: `fix: resolve crash on startup`

- **chore**: Routine tasks or maintenance  
  Example: `chore: update dependencies`

- **docs**: Documentation updates  
  Example: `docs: add API usage example`

- **style**: Code style changes (formatting, missing semicolons, etc.)  
  Example: `style: fix indentation in main.js`

- **refactor**: Code refactoring without changing behavior  
  Example: `refactor: optimize data processing logic`

- **test**: Adding or updating tests  
  Example: `test: add unit tests for user service`
